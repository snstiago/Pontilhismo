# Pontilhismo

This project is a visual particle / point animation system.

## Critical visual constraints

- Do not change the visual identity unless explicitly asked.
- Do not change colors, layout, typography, composition, point density, sliders, presets, or copy unless explicitly asked.
- Make surgical changes only.
- Before editing, explain which files you plan to change and why.
- Preserve the existing dot / particle aesthetic.
- Avoid treating dots like a locked mesh or a warped image surface.
- Dots should feel like independent particles, with local movement and variation.
- Do not introduce new animation systems, new streams, new layers, or new visual concepts unless explicitly asked.
- Do not refactor unrelated code.
- Every changed line should trace directly to the requested task.

## Workflow

- Read the relevant files before editing.
- Ask if the request is ambiguous.
- Prefer the smallest possible code change.
- After editing, explain what changed and how to verify it.
